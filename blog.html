General Overview

This blog entry discusses different interactive technologies that might be used to create a mobile, web or desktop (or laptop) based experience that could complement or support an IoT project. The technologies discussed are:

  • Processing - PDE and libraries
  • Unity Game Engine
  • HTML5 and Web Development

Processing Overview

Processing and the Processing Development Environment (PDE) are cross-platform tools for introducing people to programming. They are written in the Java language and run on the Java runtime. What does this mean? It means that Processing runs on Macos, Windows and Linux and that the way you express PDEas in Processing is heavily influenced by the ways that PDEas are expressed in Java.

The Processing PDE was used to create the Arduino PDE so if it feels familiar, this is why. Like the Arduino PDE, Processing files are called sketches and this name actually makes more sense in Processing than it did in Arduino. Each Processing program is a form of computer graphics or 'sketch.'

Processing has modes and the computer based mode (Windows, Macos, Linux) is the default. There are other modes available that can be downloaded and added to the PDE, such as Web development, using JavaScript, and Android development using an Android Java environment.

Processing on the Desktop / Laptop

Processing on a personal computer can be used to create rich visualizations and programs that communicate with external hardware (i.e., Arduino) via serial port, Raspberry Pi, motion sensors and so on.

For visualization, it is capable of 2D and 3D graphics, vPDEo playback and various forms of animation. This is possible through a wPDE variety of libraries written by members of the Processing Community.

A Processing program is called a sketch. These are written in a cut-down version of Java. It is 'cut-down' because you don't need to think about traditional encapsulation with private and public methods. Classes are supported but, for a simple sketch, are not needed. A sketch is made up of special functions:

  1. setup()
  2. draw()
  3. mousePressed()

The setup() function initializes the environment - any variables or object needed for the visualization, simulation or graphics. The draw() function is repeatedly called by the Processing environment to draw the next frame.

Example Sketch - RadialGradient - Supplied with Processing

/**
* Radial Gradient. 
* 
* Draws a series of concentric circles to create a gradient 
* from one color to another.
*/

int dim;

void setup() {
  size(640, 360);
  dim = width/2;
  background(0);
  colorMode(HSB, 360, 100, 100);
  noStroke();
  ellipseMode(RADIUS);
  frameRate(1);
}

void draw() {
  background(0);
  for (int x = 0; x <= width; x+=dim) {
    drawGradient(x, height/2);
  } 
}

void drawGradient(float x, float y) {
  int radius = dim/2;
  float h = random(0, 360);
  for (int r = radius; r > 0; --r) {
    fill(h, 90, 90);
    ellipse(x, y, r, r);
    h = (h + 1) % 360;
  }
}

Observations on Desktop Sketch Language

Function (or method) declarations, for-loops, variable declarations and object creation (uses new operator) are all from the Java language. Note that the sketch language is translated into Java and Processing changes the normal flow of Java. If you hit a problem when developing your project, you need to find examples or ask questions from the Processing Community to get help. You can't seek help from Java documentation because Processing's Sketch files don't follow Java conventions or work in the exact same way that Java does.

The good news is that there is a lot of power in the provPDEd libraries, the documentation is quite good and the community shares a lot of very useful code.

Processing on the Web

There is a P5JS mode which uses JavaScript and HTML. No examples for P5JS mode are available from the PDE (unlike Java). Instead, you need to perform an internet search to find useful examples.

The good news is that many examples can be found at p5js.org such as this Snowflake example. 

let snowflakes = []; // array to hold snowflake objects

function setup() {
  createCanvas(400, 600);
  fill(240);
  noStroke();
}

function draw() {
  background('brown');
  let t = frameCount / 60; // update time

  // create a random number of snowflakes each frame
  for (let i = 0; i < random(5); i++) {
    snowflakes.push(new snowflake()); // append snowflake object
  }

  // loop through snowflakes with a for..of loop
  for (let flake of snowflakes) {
    flake.update(t); // update snowflake position
    flake.display(); // draw snowflake
  }
}

// snowflake class
function snowflake() {
  // initialize coordinates
  this.posX = 0;
  this.posY = random(-50, 0);
  this.initialangle = random(0, 2 * PI);
  this.size = random(2, 5);

  // radius of snowflake spiral
  // chosen so the snowflakes are uniformly spread out in area
  this.radius = sqrt(random(pow(width / 2, 2)));
  
  this.update = function(time) {
    // x position follows a circle
    let w = 0.6; // angular speed
    let angle = w * time + this.initialangle;
    this.posX = width / 2 + this.radius * sin(angle);

    // different size snowflakes fall at slightly different y speeds
    this.posY += pow(this.size, 0.5);

    // delete snowflake if past end of screen
    if (this.posY > height) {
      let index = snowflakes.indexOf(this);
      snowflakes.splice(index, 1);
    }
  };

  this.display = function() {
    ellipse(this.posX, this.posY, this.size);
  };
}
